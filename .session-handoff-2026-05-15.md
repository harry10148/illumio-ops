## 並行執行的關鍵收穫

1. **git worktree + mega-subagent 模式**奏效：
   - Phase 2.2 + 3.3 並行（dashboard vs report，幾乎無檔案重疊）→ 一次 session merge 兩個 phase
   - Phase 1.1 + 3.1 並行（settings panel vs dashboard panel，分屬 `index.html` 不同 panel）→ 同上
   - 必須用 i18n key prefix 隔離（`gui_settings_*` vs `gui_dashboard_*`）避免 JSON merge conflict

2. **Pragmatic threshold-bump 策略**：subagent 跑大 task 時容易卡在 perfect migration；改成「migrate 2-3 top instances + lock 剩餘 count 為 baseline」可避免無限 stuck

3. **Subagent 連續執行多個 task** 時容易 context overflow；單一 mega-subagent 上限約 7-8 個 small task；若失敗，分散派 + 重試成功
